## 🚀 Sobre o Projeto

O **LinkBio Brasil** é uma plataforma completa para criação de páginas de links personalizadas. Desenvolvido com tecnologias modernas, oferece uma experiência fluida e profissional para usuários que desejam compartilhar múltiplos links de forma organizada.

### Por que usar o LinkBio Brasil?

- ✅ **Totalmente em Português** - Interface e documentação 100% em português
- ✅ **Grátis e Open Source** - Código disponível para customização
- ✅ **Fácil de Usar** - Interface intuitiva e moderna
- ✅ **Análises de Cliques** - Acompanhe o desempenho dos seus links
- ✅ **Personalização** - Adicione ícones, descrições e reorganize seus links
- ✅ **Responsivo** - Funciona perfeitamente em qualquer dispositivo
- ✅ **Modo Escuro** - Suporte nativo para tema escuro
- ✅ **Autenticação Múltipla** - Login com Google, GitHub ou Email/Senha

## 📦 Instalação

### Pré-requisitos

Antes de começar, certifique-se de ter instalado:

- **[Node.js](https://nodejs.org/)** (versão 18.x ou superior)
- **[npm](https://www.npmjs.com/)** ou **[yarn](https://yarnpkg.com/)**
- **[Git](https://git-scm.com/)** (opcional)

### Passo 1: Clone o repositório

```bash
git clone https://github.com/seu-usuario/linkbio-brasil.git
cd linkbio-brasil
```

### Passo 2: Instale as dependências

```bash
npm install
```

### Passo 3: Configure as variáveis de ambiente

Crie um arquivo `.env` na raiz do projeto:

```env
# Database
DATABASE_URL="prisma+postgres://localhost:51213/?api_key=..."

# NextAuth
NEXTAUTH_SECRET="sua-chave-secreta-aqui"
NEXTAUTH_URL="http://localhost:3000"

# OAuth (opcional - para login social)
GOOGLE_CLIENT_ID="seu-google-client-id"
GOOGLE_CLIENT_SECRET="seu-google-client-secret"

GITHUB_CLIENT_ID="seu-github-client-id"
GITHUB_CLIENT_SECRET="seu-github-client-secret"

# Stripe (opcional - para pagamentos)
STRIPE_SECRET_KEY="sk_test_..."
STRIPE_WEBHOOK_SECRET="whsec_..."
```

### Passo 4: Inicie o banco de dados

```bash
# Inicia o PostgreSQL local do Prisma
npx prisma dev
```

### Passo 5: Execute as migrações

```bash
npx prisma migrate dev
```

### Passo 6: Gere o cliente Prisma

```bash
npx prisma generate
```

### Passo 7: Inicie o projeto

```bash
npm run dev
```

O projeto estará disponível em [http://localhost:3000](http://localhost:3000)

## 🔌 API Endpoints

### Autenticação

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| GET | `/api/auth/[...nextauth]` | Rotas do NextAuth |
| POST | `/api/auth/signup` | Criar nova conta |

### Links

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| GET | `/api/links` | Listar todos os links do usuário |
| POST | `/api/links` | Criar novo link |
| PATCH | `/api/links/[id]` | Atualizar link |
| DELETE | `/api/links/[id]` | Deletar link |

### Perfil

| Método | Endpoint | Descrição |
|--------|----------|-----------|
| GET | `/api/profile` | Buscar perfil do usuário |
| PATCH | `/api/profile` | Atualizar perfil do usuário |

### Exemplo de Uso da API

#### Criar um novo link

```bash
curl -X POST http://localhost:3000/api/links \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Meu Portfolio",
    "url": "https://meusite.com",
    "description": "Veja meus projetos",
    "icon": "🎨"
  }'
```

#### Atualizar um link

```bash
curl -X PATCH http://localhost:3000/api/links/abc123 \
  -H "Content-Type: application/json" \
  -d '{
    "title": "Novo Título",
    "isActive": false
  }'
```

#### Deletar um link

```bash
curl -X DELETE http://localhost:3000/api/links/abc123
```

## 🗄️ Modelo de Dados

### User (Usuário)

```prisma
model User {
  id            String       @id @default(cuid())
  email         String       @unique
  name          String?
  username      String       @unique
  password      String?
  image         String?
  bio           String?
  createdAt     DateTime     @default(now())
  updatedAt     DateTime     @updatedAt
  links         Link[]
  subscription  Subscription?
  sessions      Session[]
  accounts      Account[]
}
```

### Link

```prisma
model Link {
  id          String    @id @default(cuid())
  title       String
  url         String
  description String?
  icon        String?
  position    Int
  isActive    Boolean   @default(true)
  clicks      Int       @default(0)
  createdAt   DateTime  @default(now())
  updatedAt   DateTime  @updatedAt
  clickLogs   Click[]
  userId      String
  user        User      @relation(...)
}
```

### Click (Registro de Clique)

```prisma
model Click {
  id        String    @id @default(cuid())
  createdAt DateTime  @default(now())
  userAgent String?
  referrer  String?
  country   String?
  city      String?
  linkId    String
  link      Link      @relation(...)
}
```

### Subscription (Assinatura)

```prisma
model Subscription {
  id                  String    @id @default(cuid())
  status              String    @default("active")
  plan                String    @default("free")
  stripeCustomerId    String?
  stripePriceId       String?
  currentPeriodEnd    DateTime?
  createdAt           DateTime  @default(now())
  updatedAt           DateTime  @updatedAt
  userId              String    @unique
  user                User      @relation(...)
}
```

## 🐛 Solução de Problemas

### Erro: "Cannot find module '@prisma/client'"

```bash
npm install @prisma/client
npx prisma generate
```

### Erro: "Database connection failed"

Certifique-se de que o banco de dados está rodando:

```bash
npx prisma dev
```

### Erro: "NextAuth configuration error"

Verifique se as variáveis de ambiente estão configuradas corretamente no arquivo `.env`.

### Erro: "Port 3000 is already in use"

```bash
# Usar outra porta
PORT=3001 npm run dev
```

### Erro: "Migration failed"

```bash
# Resetar o banco de dados (ATENÇÃO: isso apaga todos os dados)
npx prisma migrate reset
```
